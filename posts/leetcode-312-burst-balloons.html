Leetcode 312: Burst Balloons

Published on
May 26, 2021
· 5 mins
LeetcodeDynamic Programming

https://leetcode.com/problems/burst-balloons/

Problem statement

You are given n balloons, indexed from 0 to n - 1. Each balloon is painted with a number on it represented by an array nums. You are asked to burst all the balloons.

If you burst the ith balloon, you will get nums[i - 1] * nums[i] * nums[i + 1] coins. If i - 1 or i + 1 goes out of bounds of the array, then treat it as if there is a balloon with a 1 painted on it.

Return the maximum coins you can collect by bursting the balloons wisely.

Working with example:

Input: nums = [3,1,5,8]
Output: 167
Explanation:
nums = [3,1,5,8] --> [3,5,8] --> [3,8] --> [8] --> []
coins =  3*1*5    +   3*5*8   +  1*3*8  + 1*8*1 = 167

The balloon are popped in this sequence
1. Pop 1
2. Pop 5
3. Pop 3
4. Pop 8

Brute force solution

If we go by the naive solution, we can assign each number with sequence number it will be popped. This solution will yield time complexity of O(n!).

Attempt to break it into sub-problems

Can we solve this via the Divide and Conquer method by dividing the list into sub-lists and solve them individually. At each step, we select a balloon to be popped, the remaining elements will form sub-sequence of the original list.

To illustrate this, given that we have a list of [3, 1, 5, 8]

If we pop off 1 first
We have to deal with [3, 5, 8] next

If we use such strategy, our next subproblem will going to be [3, 5, 8]. We don't exactly break the list into two sublists [3] and [5, 8] and deal with them individually. Each sub-list cannot represent a subproblem. Instead, two parts of sublists are "connected" and contiguous in a sense. Our next step will be obtaining max gain from this merged sub-lists. We are going to exploring all 2n2^n2n possible sub-sequences whicih is undesirable, this will be an exponential growth complexity solution.

More optimized way of solving sub-problems

How about we take a different approach, instead of choosing which balloon to be popped first, we reverse our thinking by selecting which balloon to be popped last.

With a list of [3, 1, 5, 8,], we pick 1 as the balloon to popped last,

If we choose 1 as the last balloon to be popped, we are not dealing with sub-lists [3, 5] and [8] as they are contiguous. We can formulate the next problem as finding the maximum gain each from [3, 5] and [8]. The maximum gain of [3, 5] and [8] are formulated not to be depending on each other (unlike the previous section). This makes the sub-list can be solved individually.

We need to introduce some auxiliary setting to help deal with boundary balloons,

We pad the list with balloon 1. Since these are paddings, we will not burst these balloons.

Using this concept, we can formulate as follow:

  • To compute the max gain, DP of a list, we have to compute: max(DP(left_sublist) + (balloon that we remove at last * boundary elements) + DP(right_sublist))

  • This computation is repeated for each sublist.

  • Boundary element lies outside of the list, it can be padding OR element excluded from the sublist

When we choose a balloon to be popped last, the calculation takes the balloons at far left and right (boundary, that is not included in the list) into acccount

Notice that the calculation will at least have 3 elements, if we are evaluating sub-list of size 2, we treat it as our base case and have max gain of 0. To illustrate this, let say we have a list of zero element:

The auxilliary list will probably looks something like this. Since there's no element, the max gain will be zero.

When we compute the max gain of a list, we will try all possible choices of balloon to be popped last

When we burst the balloon at index k, we can get:

  • DP[i,k] (Maximum coins collected by bursting balloons[(i+1)…(k-1)]) +

  • DP[k,j] (Maximum coins collected by bursting balloons[(k+1)…(j-1)]) +

  • nums[i] nums[k] nums[j]

Let's get into the programming

Flow of the program

  1. Define A[] = {1, nums[0], nums[1], …, nums[n-1], 1} of size (n+2). Let these represent the balloons.

  2. Define an DP matrix of size (n+2)*(n+2), such that DP[i][j] (j > i) represents the maximum coins collected by bursting balloons [(i+1)…(j-1)].

  3. (Base Case) DP[i][i+1] = 0 because we have no ballons to burst.

  4. DP[i][j] = Max ( DP[i][k] + DP[k][j] + (nums[i]nums[k]nums[j]), where (i+1)≤ k ≤ (j-1) ).

  5. Final answer is DP[0, n+1].

Depends on the approach that we use, if we going for top-down, we would solve the problem from the entire list. If we take the bottom-up approach, we can solve from sublist from size 2 and onwards.

Solution:

class Solution {
    public int maxCoins(int[] nums) {
        int size = nums.length;
        // auxilliary array
        int[] arr = new int[size+2];
        arr[0] = 1;
        arr[size+1] = 1;
        
        for (int i=1; i<=size; i++) {
            arr[i] = nums[i-1];
        } 
            
        int[][] dp = new int[size+2][size+2];
                    
        // i, j are the boundary indices, in between them are the "true list"
        // iterate through the size of sublist
        for (int s=2; s<=(size+2); s++) { 
            // iterate through the left boundary
            
            for (int i=0; i<size+2-s+1; i++) { 
                // j is index of next element that we remove
                
                for (int j=i+1; j<i+s-1; j++) {
                    if (s==2) {
                        dp[i][i+s-1] = 0;
                    } else {
                        dp[i][i+s-1] = Math.max(dp[i][i+s-1], 
                                                dp[i][j] + dp[j][i+s-1] + arr[i]*arr[j]*arr[i+s-1]);
                    }   
                }
                
            }
            
        }

        return dp[0][size+1];
        
    }
    
}

Runtime Analysis

The DP table has O(N²) entries. To compute each entry, we are looking at all the possible choices for the last balloon to be burst and taking the maximum one. There are O(N) choices in the worst case at each step and for each choice computing the coins collected is an O(1) operation.Thus, computing each entry of DP[][] takes O(N) * O(1) = O(N) time.Hence, the overall runtime is O(N²) x O(N) = O(N³).
