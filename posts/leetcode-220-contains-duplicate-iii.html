Understanding problem statement

This problem specifies, for each element, by definition its surroundings (elements bounded by k window size), if any of the difference (between itself and its surroundings) yield a value of not greater than t, return true, otherwise return false.

Constructing the solution

We might be tempted to iterate through each element, going through its surroundings to check for the conditions. However, this will arrive to a solution that has complexity of O(n*k).

Some speedup can be achieved if we get the sorted array and skip those elements that yields difference more than t.

Intuition: If we sort the array, we would effectively know the region (indicated by left and right pivots) that would satisfy t difference criteria. Let say we left and right pivots is yielding difference greater than t, since the right pivot will be pointing at a greater value, by moving the left pivot to right, we might miminize the difference hence satisfy the condition.

After the less than t difference is satisfied, we check the corresponding index in the original array, making sure that the distance between the indices in below k.

class Solution {
    public boolean containsNearbyAlmostDuplicate(int[] nums, int k, int t) {
        // k indicates the window size
        // t indicates the difference that it must satisfy
       
        Integer[] ord = new Integer[nums.length];
        
        
        for (int i=0; i<nums.length; i++) {
            ord[i] = i;
        }
        
        Arrays.sort(ord, (a,b)->Integer.compare(nums[a], nums[b]));
        

        int left = 0;
        int right = 1;
        
        while (right<nums.length) {        
            // this will get us the actual value nums[ord[i]]
            
            // move the left pivot if it produces diff larger than t 
            if ((long) nums[ord[right]]-nums[ord[left]] > t) {
                left++;
                // left pivot cannot be at the same position as right pivot
                if (left == right) right++;
                continue; // we only move the left pivot once, it might still not satisfy the condition
            }
            
            // after the above, we ensure that the region between left and right pivot satisfy the t condition
            // hence what we need to do next is to while fixing right pivot
            // find any index from the left that satisfy k condition
            
            // we introduce another temp pivot that moves from left pivot to right pivot
            int temp = left;
            while(temp < right) {
                if (Math.abs(ord[temp]-ord[right]) <= k) return true;
                temp++;
            }
            
            right++;
            
        }
        return false;
    }

    
}
